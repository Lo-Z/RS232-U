## Welcome to RS232-U V3

The easy way to test, troubleshoot, monitor, and control RS-232 communication in the field.

The **RS232-U V3** is a portable RS-232 diagnostic and field utility tool designed for installers, technicians, integrators, and service professionals.

The RS232-U V3 can be used to send predefined device control commands, test common DB9 cable configurations, verify loopback operation, check communication paths, send and receive serial data through the RS232-U Web Utility, monitor inline RS-232 traffic, save command sets, and update firmware through a supported desktop browser.

## 3. Important Safety Information

Read this section before using or charging the RS232-U V3.

3.0 Use the RS232-U V3 only with RS-232 equipment and approved accessories.

3.1 Use only a suitable USB-C cable and standard USB power source for charging.

3.2 Do not use a damaged USB-C cable, damaged charger, or loose power connection.

3.3 Do not use the device if the enclosure, USB-C connector, DB9 connectors, terminal block connector, display, or buttons are visibly damaged.

3.4 Disconnect USB-C power immediately if the device:

* Becomes unusually hot
* Develops an unusual odor
* Shows signs of battery swelling
* Shows signs of enclosure separation
* Shows signs of physical damage

3.5 The RS232-U V3 contains an internal rechargeable LiPo battery.

* Do not puncture, crush, bend, or damage the battery.
* Do not use, charge, or attempt to repair a swollen, leaking, punctured, crushed, or otherwise damaged LiPo battery.
* Do not leave the device charging unattended for extended periods.
* Do not open, modify, or service the internal battery unless qualified to do so.

3.6 Store the device away from excessive heat, moisture, and direct sunlight.

3.7 Battery servicing should only be performed by qualified personnel.

## 4. Charging and Power

### 4.0 Battery Type

The RS232-U V3 uses an internal rechargeable **2,000mAh Lithium Polymer LiPo battery**.

For the best experience, use the USB-C cable that comes with your device.

### 4.1 Charging Procedure

* Connect a suitable USB-C cable to the RS232-U V3.
* Connect the other end of the cable to a standard USB power source.
* Allow the battery to charge fully before extended field use.
* Disconnect the USB-C cable when charging is complete.

### 4.2 Charging Notes

* The RS232-U V3 automatically powers on when a USB-C cable is connected.
* Typical 0–100% charging time is approximately **5–6 hours**.
* Charging time may vary depending on battery condition, USB power source capability, charge level, and system activity.
* An internal blue LED illuminates through the LED grille while the battery is charging.
* Charging is complete when the blue LED turns off and the display reads **100%**.
* If the battery reaches 0%, the system will display a low-battery screen and temporarily prevent normal use until an adequate charge level has been restored.

### 4.3 Battery Runtime

Battery life varies with screen brightness, serial activity, Web Utility use, and power settings.

* Typical battery runtime is up to **18 hours**, depending on usage conditions.
* Internal idle runtime testing exceeded **21 hours and 46 minutes** with the RS232-U V3 powered on and no active serial use.

### 4.4 Power On

* Press and hold the Power Button.
* Release the button when the RS232-U V3 starts.
* Wait for the main menu to appear.

### 4.5 Power Off

* Press and hold the Power Button for approximately 3 seconds.
* When the power-off menu appears, select the power-off option.
* Confirm the selection.

### 4.6 Sleep Timer

The sleep timer automatically powers off the RS232-U V3 after a selected period of inactivity.

Pressing any navigation button resets the sleep timer.

Available sleep timer options:

* Off
* 30 Seconds
* 1 Minute
* 1.5 Minutes
* 2 Minutes

## 7. Cable Test

![Cable Test Mode](docs/images/cable-test-mode.png)

### 7.0 Cable Test Feature

Cable Test Mode verifies supported three-wire DB9 RS-232 cable wiring configurations by checking the expected TX, RX, and Ground signal paths.

### 7.1 Supported Cable Tests

The RS232-U V3 can directly test:

* Male-to-male straight-through cables
* Male-to-female straight-through cables
* Female-to-female straight-through cables
* Male-to-female null-modem cables

### 7.2 Basic Procedure

* Identify the connector types on the cable under test.
* Determine whether the cable is expected to be straight-through or null-modem.
* Connect each end of the cable to the appropriate DB9 ports on the RS232-U V3.
* Select **Cable Test** from the main menu.
* Select the matching cable type.
* Select **Test Cable**.
* Review the displayed result.

### 7.3 Test Results

* **Successful** — The expected TX, RX, and Ground signal paths were detected successfully.
* **Failed** — One or both signal paths did not send or receive data correctly. Check cable wiring, connector seating, and expected pinout.

### 7.4 Straight-Through Cables

A straight-through cable connects matching pin numbers between both ends.

```text
Pin 2 -> Pin 2
Pin 3 -> Pin 3
Pin 5 -> Pin 5
```

Use one DTE / Standard port and one DCE / Null port when testing a straight-through cable.

### 7.5 Null-Modem Cables

A null-modem cable is a crossover cable. It swaps the TX and RX paths between both ends.

```text
Pin 2 -> Pin 3
Pin 3 -> Pin 2
Pin 5 -> Pin 5
```

Use the **Male DTE / Standard** and **Female DTE / Standard** ports when testing a male-to-female null-modem cable.

### 7.6 Cable Test Notes

* Null cables can only be tested directly through the Male DTE / Standard and Female DTE / Standard ports due to their pinout.
* Male-to-male and female-to-female null-modem cables cannot be tested directly using the current RS232-U V3 port layout.
* Testing these cable types may require an appropriate straight-through gender changer.
* The 3-pin terminal block connector is internally connected to the DTE / Standard ports and can be used wherever a DTE / Standard port is used.
* Cable Test Mode may not fully characterize custom-wired cables, manufacturer-specific cables, or cables that use additional RS-232 handshake pins.

## 9. Loopback Test

### 9.0 Loopback Test Feature

Loopback Test verifies that transmitted RS-232 data can return to the RS232-U V3 through the connected cable or signal chain.

### 9.1 Basic Procedure

* Connect either of the RS232-U V3’s DTE / Standard DB9 ports or the 3-pin terminal block connector at the beginning of the RS-232 cable or signal chain.
* At the far end of the cable or signal chain, temporarily connect TX to RX.
* Select **Loopback Test** from the main menu.
* Select **Start**.
* Review the displayed result.

### 9.2 Test Results

* **Successful** — The transmitted data returned through the RX path successfully.
* **Failed** — The transmitted data was not received. Check cable connections, TX/RX termination, and signal path.

A successful result confirms that the transmit and receive paths are functioning through the connected cable or signal chain.

## 10. Coms Check

### 10.0 Coms Check Feature

Coms Check sends a generic RS-232 probe to the connected device in an attempt to receive a response.

Not all RS-232 devices respond to generic probes.

Coms Check is intended as a quick first-step troubleshooting tool, not a definitive communication test.

### 10.1 Available Probes

* `?`
* `GET`
* Carriage Return
* `0x00`

### 10.2 Basic Procedure

* Connect either of the RS232-U V3’s DTE / Standard DB9 ports or the 3-pin terminal block connector to the RS-232 device under test.
* Select **Coms Check** from the main menu.
* Select a generic probe.
* Review the displayed result.

### 10.3 Important Notes

* A failed response does not necessarily indicate a bad connection, incorrect wiring, or poor termination.
* Some devices only respond to specific commands defined by the manufacturer.
* Confirm the required baud rate, serial format, and pinout when troubleshooting a specific device.
* Use Coms Check as an initial troubleshooting step before moving to Terminal Mode for more detailed testing.

### 11.6 Sending ASCII Commands

ASCII mode sends the command payload as readable text.

Use ASCII mode when the device protocol shows commands as plain text characters.

Examples:

```text
PWR ON
GET
?
```

Escape sequences may be used when supported by the Web Utility, such as `\r` for carriage return.

### 11.7 Sending HEX Commands

HEX mode sends the command payload as hexadecimal byte values.

Use HEX mode when the device protocol provides commands as byte values instead of plain text.

Examples:

```text
8C 00
8C,00
0x8C,0x00
\x8C\x00
```

## 12. Inline Monitor

![Inline Monitor Standard Connection](docs/images/inline-monitor-standard.png)

![Inline Monitor Null Connection](docs/images/inline-monitor-null.png)

### 12.0 Inline Monitor Feature

Inline Monitor allows the RS232-U V3 to be placed between a controller and a connected RS-232 device.

While connected inline, the RS232-U V3 passes serial traffic between both devices and displays communication activity for the user to view.

Inline Monitor is useful when verifying:

* Whether a controller is sending commands
* Whether a device is responding
* Whether RS-232 traffic is present on an existing control path

### 12.1 Basic Operation

Inline Monitor does not create new commands.

It monitors and relays the RS-232 communication already taking place between the connected controller and device.

Depending on the wiring requirement, Inline Monitor may use the DTE / Standard / 3-pin terminal block connector and DCE / Null signal paths to sit between both ends of the RS-232 connection.

Use the connection diagram that matches the selected Inline Monitor mode.

### 12.2 Basic Procedure

* Identify the controller and the RS-232 device being monitored.
* Determine the required Inline Monitor connection type.
* Connect the controller and device to the RS232-U V3 using the appropriate Inline Monitor connection diagram.
* Select **Inline Monitor** from the RS232-U V3 main menu.
* Start Inline Monitor.
* View the RS-232 communication activity on the RS232-U V3.
* Review or export saved Inline Monitor logs through the RS232-U Web Utility when needed.

### 12.3 Inline Monitor Logs

Each Inline Monitor session is automatically saved using a sequential log filename.

Examples:

```text
LOG0001
LOG0002
LOG0003
```

Saved logs can be renamed manually.

### 12.4 Web Utility Inline Monitor Controls

The Inline Monitor tab includes:

* Connect Device
* Refresh
* View Log
* Rename
* Download
* Remove

The Inline Monitor tab also shows microSD card storage information, including total, used, and free storage.

### 12.5 Important Notes

* Inline Monitor is intended to monitor RS-232 communication between two connected devices.
* The selected connection type must match the wiring path being monitored.
* The RS232-U V3 must be placed inline between the controller and the RS-232 device.
* If no traffic appears, confirm that the controller is actively sending commands.
* If responses are missing, confirm the connected device is powered on, configured correctly, and using the expected baud rate and serial format.
* Some systems only transmit when a command, button press, automation event, or scheduled action occurs.

## 13. Saved Commands

![Saved Commands](docs/images/saved-commands.png)

### 13.0 Saved Commands Feature

Saved Commands allows RS-232 command sets created in the RS232-U Web Utility to be saved to the RS232-U V3 microSD card, organized into folders, recalled later, and used directly from the RS232-U V3 without reconnecting to Terminal Mode.

This feature is useful for storing:

* Known-working RS-232 command sets
* Common device control commands
* Frequently used troubleshooting commands
* Project-specific command groups
* Room-specific command groups

### 13.1 Creating Saved Commands

Saved Commands are created through Terminal Mode in the RS232-U Web Utility.

To create a saved command set:

* Connect the RS232-U V3 to a computer using a USB-C data cable.
* On the RS232-U V3, select **Terminal Mode** from the main menu.
* Open the RS232-U Web Utility in a supported browser.
* Open the **Terminal Mode** tab.
* Select **Connect Device**.
* Enter the desired command names and command payloads.
* Select **Save Command** to save the command set to the RS232-U V3 microSD card.

### 13.2 Using Saved Commands on the RS232-U V3

To use a saved command set:

* Select **Saved Commands** from the RS232-U V3 main menu.
* Browse to the desired folder or saved command file.
* Open the saved command set.
* Select the desired command.
* Send the command to the connected RS-232 device.

### 13.3 Saved Commands Web Utility Controls

The Saved Commands tab includes:

* Refresh
* New Folder
* Load Into Terminal
* Remove

The Saved Commands tab may also display:

* Current folder path
* Contents of the selected folder
* microSD card storage information

### 13.4 Organizing Saved Commands

Saved commands can be organized by:

* Manufacturer
* Device model
* Project or site name
* Room name
* Command type

### 13.5 Important Notes

* The RS232-U V3 must be connected in Terminal Mode for the Web Utility to access saved commands on the microSD card.
* Saved commands are intended to store known-working RS-232 command sets for later use.
* The selected baud rate, serial format, and command formatting must still match the connected device.
* Removing a saved command file or folder permanently deletes it from the RS232-U V3 microSD card.
* The Load Into Terminal function is useful for quickly recalling previously saved commands for additional testing or editing.

## 15. Connectors and Terminals

![RS232-U V3 Port Layout](docs/images/rs232-u-v3-port-layout.png)

### 15.0 Connections

The RS232-U V3 includes:

* 1 USB-C port
* 1 Male DTE / Standard DB9 port
* 1 Female DTE / Standard DB9 port
* 1 Male DCE / Null DB9 port
* 1 Female DCE / Null DB9 port
* 1 3-pin terminal block connector
* microSD card storage

### 15.1 USB-C

USB-C is used for:

* Charging
* Web Utility communication
* Firmware updates

Use a USB-C cable that supports data transfer.

The USB-C cable included with the RS232-U V3 is recommended.

### 15.2 RS-232 Port Types

The RS232-U V3 includes two types of RS-232 ports:

* DTE / Standard ports
* DCE / Null ports

DTE / Standard and DCE / Null describe the transmit and receive pin assignments of the ports.

They do not describe whether the connector is male or female.

Connector gender only describes the physical DB9 connector type.

### 15.3 Port Pinouts

| Port Type      | Location            | Pin 2 | Pin 3 |  Pin 5 |
| -------------- | ------------------- | ----: | ----: | -----: |
| DTE / Standard | Top of device       |    RX |    TX | Ground |
| DCE / Null     | Left side of device |    TX |    RX | Ground |

### 15.4 Terminal Block Connector

The 3-pin terminal block connector is internally connected to the DTE / Standard signal path.

It can be used wherever a DTE / Standard port is used.

## 16. Firmware Updates

### 16.0 Firmware Update Feature

The RS232-U V3 supports browser-based firmware updates through the RS232-U Web Utility.

Firmware updates are performed from the Firmware Updater tab using an approved `.bin` firmware file.

### 16.1 Before Updating

Before starting a firmware update:

* Charge the RS232-U V3 battery to at least 30%.
* Use a reliable USB-C data cable.
* Close any other application or browser tab that may already be using the RS232-U V3 serial port.
* Confirm that the firmware file being used is intended for the RS232-U V3.

### 16.2 Firmware File Format

Firmware updates are provided as `.bin` files.

The filename follows this general format:

```text
rs232u_version_releasedate.bin
```

Example:

```text
rs232u_v2.0.0_20260530.bin
```

### 16.3 Firmware Update Procedure

* Connect the RS232-U V3 to the computer using a USB-C data cable.
* Open the RS232-U Web Utility in a supported browser.
* Open the **Firmware Updater** tab.
* Select **Choose File**.
* Choose the approved `.bin` firmware file.
* Select **Connect Device**.
* Choose the RS232-U V3 serial device when prompted by the browser.
* Select **Start Flashing**.
* Wait for the firmware update process to complete.
* Confirm that the RS232-U V3 restarts normally.

### 16.4 Firmware Update Warning

**Do not disconnect power, remove the USB-C cable, close the browser tab, or power off the computer while firmware is being written.**

Interrupting power or disconnecting USB-C during a firmware update may cause the RS232-U V3 to stop booting correctly and require recovery service.

Do not unplug the device until the update process has fully completed.

### 16.5 After Updating

After a successful firmware update, it may be necessary to power the RS232-U V3 off and back on before reconnecting to Terminal Mode or other Web Utility features.

## 17. Troubleshooting

| Issue                                                | Recommended Action                                                                                                                                                                                                                                                                               |
| ---------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Device does not power on                             | Connect the RS232-U V3 to USB-C power and allow the battery to charge. Press and hold the Power Button. If the device does not respond, try a different USB-C cable and USB power source.                                                                                                        |
| Battery does not appear to charge                    | Confirm that the USB-C cable is fully inserted. Try a different USB-C cable and USB power source. Confirm that the blue charging LED illuminates when USB-C power is connected.                                                                                                                  |
| Device becomes unusually hot while charging          | Disconnect USB-C power immediately. Stop using the device until it has been inspected. Do not continue charging if the enclosure separates, the battery swells, or an unusual odor is present.                                                                                                   |
| Device powers off unexpectedly                       | Check the battery charge level and configured sleep timer setting. Connect USB-C power and recharge the battery if necessary.                                                                                                                                                                    |
| Device Control command does not work                 | Confirm that the correct device manufacturer is selected. Verify that the selected command is supported by the connected device model. Check the RS-232 cable type, baud rate, serial format, and pinout using the manufacturer’s documentation. Try Terminal Mode for detailed troubleshooting. |
| Cable Test fails                                     | Confirm that both ends of the cable are fully inserted. Confirm that the selected cable type matches the cable under test. Verify that the cable is connected to the appropriate DTE / Standard and DCE / Null ports. Test with a known-good cable.                                              |
| Loopback Test fails                                  | Confirm that the RS232-U V3 is connected at the beginning of the RS-232 cable or signal chain. Confirm that TX and RX are connected together at the far end of the cable or signal chain.                                                                                                        |
| Coms Check does not receive a response               | Confirm that the RS232-U V3 is connected to the correct RS-232 port or terminal block connection. Try each available generic probe. Confirm baud rate, serial format, and pinout. Use Terminal Mode to send a manufacturer-specific command.                                                     |
| Terminal Mode does not receive a response            | Confirm that the RS232-U V3 is set to Terminal Mode from the device main menu. Confirm that the Web Utility is connected. Verify RS232 port, baud rate, serial configuration, send format, and command termination.                                                                              |

Confirm that the controller is actively sending commands.                                                                                                                                   |

Confirm that saved command files are located in the expected folder.                                                                                                                                      |

Load the saved command into Terminal Mode and test it manually if needed.                                                                                           |
| Web Utility cannot find the RS232-U V3               | Confirm that the USB-C cable supports data transfer. Disconnect and reconnect the cable. Close other applications using the serial port. Refresh the Web Utility and select Connect Device again.                                                                                                |

Reconnect the USB-C cable, close other applications using the serial port, and repeat the update procedure.                                                                           |

## 18. Care and Storage

To help protect the RS232-U V3 and maintain reliable operation:

* Store the device in a cool, dry location when not in use.
* Avoid leaving the device inside a hot vehicle, in direct sunlight, near heat sources, or in areas with excessive moisture.
* Do not expose the RS232-U V3 to excessive impact, crushing force, liquid exposure, or heavy objects during storage.
* Inspect the USB-C port, DB9 connectors, terminal block connector, display, buttons, and enclosure periodically for visible damage.
* Keep the USB-C port, DB9 connectors, and terminal block connector free from dust, debris, and foreign material.
* Recharge the battery periodically during long-term storage.
* Do not store the device fully discharged for extended periods.
* Do not use harsh chemicals, solvents, or abrasive cleaners on the enclosure, display, or connectors.
* Clean the exterior only with a soft, dry cloth when needed.

Do not continue using or charging the device if the enclosure begins to separate, the battery appears swollen, the device becomes unusually hot, or an unusual odor is present.

Contact support if the RS232-U V3 shows signs of physical damage, abnormal battery behavior, charging issues, or connector damage.
